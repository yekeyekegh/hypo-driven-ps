# 交付单模板(Diagnostician → Reviewer)

每轮两个评审节点,Diagnostician 各填一张,Reviewer 照单核对(读懂 test、查对错与完整性、重跑)。
照填以避免遗漏;字段不适用就写「N/A」并说明,不要删字段。

---

## 诊断交付单(节点1)

- **轮次**:第 N 轮
- **验证的 Hypo**:H? — <描述>
- **确诊判据**:<本轮要验的那条判据>
- **结论**:判据成立(真) / 判据证伪(假) / 无法判断
  - 若「无法判断」:哪条判据测不了、为何
- **测试清单**(每个 test 一行):
  | test 路径 | 测什么(一句话) | 运行命令 | 结果(红/绿) |
  |---|---|---|---|
  | docs/hypo-driven-ps/<topic>-probes/… | … | … | … |
- **完整性论证**:为何这组 test 足以判定该判据(覆盖了哪些情形、为何没有漏)。
- **证据**:关键命令 + 完整输出要点 + 退出码 + 复现步骤。

---

## 修复交付单(节点2)

- **轮次**:第 N 轮 · **本次为第 K 次修复尝试(K ≤ 3)**
- **确诊的 Hypo**:H? — <描述>
- **修复成本评估**:小修 / 成本过大(命中信号:<哪条>;成本过大者本不应走到这步,除非用户决策立即修)
- **修复方案 + 改了哪些文件**:<摘要 + 文件清单>
- **测试清单**:
  | test 路径 | 类型(复现/回归) | 运行命令 | 结果 |
  |---|---|---|---|
  | <进正式测试套件的路径> | 复现 | … | 修复前红 → 修复后绿 |
  | … | 回归 | … | 绿 |
- **回归核查**:重跑既有/回归套件的结果(有无弄坏别的)。
- **证据**:问题不再复现的命令 + 完整输出 + 退出码;全套测试通过的输出。
- **(若失败)失败说明**:本次为何未解决 / 引入了什么回归;若已是第 3 次 → 三次各试了什么、为何失败(learning)。
